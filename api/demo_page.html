🚀 Enhanced RAG API 演示

📍 API 服務狀態

服務地址: http://localhost:8003

Swagger 文檔: http://localhost:8003/docs

ReDoc 文檔: http://localhost:8003/redoc

🔑 測試憑證

演示 API Key: demo-api-key-123

管理員 API Key: admin-api-key-456

使用步驟:

  1. 使用 API Key 在 /auth/token 端點獲取 JWT Token
  2. 在 Swagger UI 右上角點擊 "Authorize" 按鈕
  3. 輸入 Bearer Token 格式: Bearer YOUR_JWT_TOKEN
  4. 開始測試各種 API 功能！

📚 API 端點總覽

GET / - API 根端點（獲取基本信息）
GET /health - 系統健康檢查
POST /auth/token - 生成訪問令牌
POST /chat - 智能問答（帶對話記憶）⭐
POST /upload - 上傳文件到知識庫
GET /knowledge-base - 獲取知識庫狀態
DELETE /knowledge-base/files/{id} - 刪除知識庫文件
GET /conversations - 獲取對話記錄
GET /conversations/stats - 獲取對話統計

🎯 核心功能演示

🧠

智能問答

支援前後文記憶的多輪對話，能理解上下文關係並提供準確回答。

📚

知識庫管理

支援多種格式文件上傳，自動提取內容並建立可搜索的知識庫。

🔐

安全認證

API Key + JWT Token 雙重認證，支援分級權限控制。

📝

對話記錄

完整的對話歷史追蹤，支援搜索和統計分析。

🚀 快速開始

1. 訪問 API 文檔

打開 Swagger UI 打開 ReDoc

2. 測試基本功能

先測試健康檢查端點確認服務正常運行：

curl http://localhost:8003/health

3. 獲取訪問令牌

curl -X POST "http://localhost:8003/auth/token" \
  -H "Content-Type: application/json" \
  -d '{"api_key": "demo-api-key-123"}'

4. 開始智能對話

curl -X POST "http://localhost:8003/chat" \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "question": "你好，請介紹一下你的功能",
    "include_sources": true
  }'

💡 使用技巧

  • 對話記憶: 在 /chat 端點的回應中會包含 context，將其作為下次請求的 conversation_context 可實現連續對話
  • 文件上傳: 支援 PDF、Word、Markdown、圖片等格式，系統會自動提取內容並建立索引
  • 權限控制: 演示帳號有 read+write 權限，可以測試所有功能
  • 錯誤處理: 所有 API 都有詳細的錯誤信息，幫助快速定位問題
